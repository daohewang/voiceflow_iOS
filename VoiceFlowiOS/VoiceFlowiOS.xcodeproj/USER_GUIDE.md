# VoiceFlow 键盘使用指南

## 🎯 核心问题解决方案

### 问题：权限弹窗导致键盘切换

**症状：**
- 在键盘中点击麦克风时弹出权限请求
- 授权后自动切换回系统键盘
- 需要多次授权，体验很差

**根本原因：**
iOS 系统在键盘扩展中弹出权限请求时，会自动切换回系统键盘以保证安全性。

**✅ 解决方案：在主 App 中预先授权**

---

## 📱 正确的使用流程

### 第一步：首次启动主 App

1. **打开 VoiceFlow App**（主应用，不是键盘）

2. **会自动弹出权限引导页面**，显示：
   ```
   欢迎使用 VoiceFlow
   为了提供最佳的语音输入体验，我们需要您授予一些必要的权限
   ```

3. **点击「开始设置」**

4. **点击「请求权限」**，系统会依次弹出：
   - 🎤 **麦克风权限请求** → 点击「允许」
   - 🗣️ **语音识别权限请求** → 点击「允许」

5. **确认两个权限都显示为「已授权」**

6. **点击「开始使用」**

---

### 第二步：启用键盘

1. 打开 **设置 → 通用 → 键盘 → 键盘**

2. 点击 **添加新键盘**

3. 找到并选择 **VoiceFlow**

4. 回到键盘列表，点击 **VoiceFlow**

5. **开启「允许完全访问」**（必须开启！）
   - 开关应该显示为**绿色**
   - 如果是灰色说明未开启

---

### 第三步：重启设备（重要！）

即使已经授权所有权限，iOS 的键盘系统缓存可能没有更新。

**必须执行完全重启：**
1. 长按电源键 → 滑动关机
2. 等待 10 秒
3. 重新开机

⚠️ **这一步非常关键，80% 的问题都是因为没有重启！**

---

### 第四步：使用键盘

1. 打开任意支持文字输入的 App（如备忘录）

2. 点击输入框

3. 长按地球图标 🌐，选择 **VoiceFlow**

4. 点击麦克风按钮 🎤

5. **不会再弹出任何权限请求**（因为已在主 App 中授权）

6. 开始说话，实时显示识别结果

7. 再次点击麦克风停止，文字自动插入

---

## ✅ 权限检查清单

在主 App 的**设置页**中，可以查看权限状态：

1. 打开 VoiceFlow 主 App

2. 点击底部「设置」标签

3. 在「应用信息」区域，点击「权限状态」

4. 应该看到：
   ```
   ✅ 麦克风权限: 已授权
   ✅ 语音识别权限: 已授权
   ✅ 完全访问: 已开启
   ```

如果有任何权限显示为**未授权**：
- 点击「打开系统设置」
- 手动开启对应权限
- **重启设备**

## 🔍 故障排除

### 问题 1：主 App 中没有弹出权限引导

**解决方案：**
1. 完全删除 VoiceFlow App
2. 在 Xcode 中清理项目（Shift + Cmd + K）
3. 重新安装
4. 首次打开应该会弹出引导

**临时方案：**
- 在主 App 的「设置」页面
- 点击「权限状态」
- 点击「打开系统设置」
- 手动开启权限

---

### 问题 2：授权后仍然不能录音

**可能原因：**
- 没有重启设备
- 「允许完全访问」未开启
- 权限授权给了错误的 target

**解决方案：**
1. 确认「允许完全访问」是**绿色**的
2. **完全重启设备**（不是重启 App）
3. 在主 App「设置」中确认权限状态
4. 如果还是失败，重新安装 App

---

### 问题 3：在某些 App 中无法使用键盘

**正常现象：**

iOS 在以下场景会**强制使用系统键盘**：
- ✋ 密码输入框
- ✋ 信用卡信息
- ✋ 锁屏状态
- ✋ 安全模式

这是 iOS 的安全机制，无法绕过。

---

### 问题 4：识别结果不准确

**可能原因：**
- 网络不稳定（SFSpeechRecognizer 需要联网）
- 环境噪音太大
- 说话不够清晰

**改进方法：**
1. 确保网络连接良好
2. 在安静的环境中使用
3. 说话速度适中，吐字清晰
4. 避免口音过重的方言

---

## 📊 使用技巧

### 获得最佳识别效果

1. **使用标准普通话**
   - 系统对普通话识别率最高
   - 避免使用方言或混合语言

2. **说话节奏**
   - 不要太快，也不要太慢
   - 每句话之间可以短暂停顿

3. **清晰发音**
   - 避免含糊不清
   - 避免吃字或连读

4. **网络环境**
   - 优先使用 WiFi
   - 4G/5G 也可以，但要保证信号稳定

5. **环境音**
   - 避免在嘈杂环境使用
   - 关闭周围的电视、音乐等

---

### LLM 润色功能（可选）

如果您配置了 LLM API Key：

1. 在主 App「设置」页面配置 API Key
   - OpenRouter API Key（推荐）
   - 或 DeepSeek API Key

2. 选择 LLM 服务商

3. 录音结束后，识别的文字会自动：
   - 修正标点符号
   - 优化语句流畅度
   - 修正明显的识别错误

4. 如果不需要润色：
   - 留空 API Key 即可
   - 直接插入原始识别文字

---

## 🆘 仍然无法解决？

### 收集诊断信息

1. 在 Xcode 中运行项目

2. 打开控制台（Cmd + Shift + Y）

3. 在搜索框输入：`KeyboardVM`

4. 切换到键盘并点击麦克风

5. 查看控制台输出：
   ```
   [KeyboardVM] 🔍 权限状态:
   ✓ 完全访问: 已开启/未开启
   ✓ 语音识别: 已授权/未授权
   ```

6. 复制完整的日志信息
