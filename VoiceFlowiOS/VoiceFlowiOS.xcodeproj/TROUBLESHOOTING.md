# 键盘录音问题故障排除指南

## 🔴 问题：显示"录音启动失败"，即使已开启"完全访问"

### 📋 问题症状
- 已在「设置 → 通用 → 键盘 → VoiceFlow」开启「允许完全访问」
- 点击麦克风按钮仍提示"录音启动失败"
- 错误代码：2003329396 (0x77760074)

---

## ✅ 解决方案（按顺序尝试）

### 🔧 方案 1：重启设备（成功率 80%）

iOS 的键盘扩展权限系统有缓存机制，开启"完全访问"后需要**重启设备**才能完全生效。

**步骤：**
1. 确认已开启「允许完全访问」
2. **完全关机**设备（不是重启 App！）
3. 等待 10 秒
4. 重新开机
5. 打开备忘录，切换到 VoiceFlow 键盘
6. 再次测试录音

**⭐ 这是最常见的解决方案！**

---

### 🔧 方案 2：重新安装 App（成功率 60%）

权限配置可能在首次安装时未正确初始化。

**步骤：**
1. **完全删除** VoiceFlow App
2. **重启设备**（重要！）
3. 在 Xcode 中清理项目：`Shift + Cmd + K`
4. 重新构建并安装：`Cmd + R`
5. 在设置中重新启用键盘并开启「允许完全访问」
6. **再次重启设备**
7. 测试录音

---

### 🔧 方案 3：重置键盘设置（成功率 50%）

系统键盘设置可能存在缓存问题。

**步骤：**
1. 打开「设置 → 通用 → 键盘 → 键盘」
2. **删除** VoiceFlow 键盘
3. 打开「设置 → 通用 → 转移或还原 iPhone → 还原 → 还原所有设置」
   - ⚠️ 这不会删除数据，但会重置系统设置
4. 设备重启后，重新添加 VoiceFlow 键盘
5. 开启「允许完全访问」
6. **重启设备**
7. 测试录音

---

### 🔧 方案 4：检查系统限制

某些企业设备或家长控制设置可能限制键盘权限。

**检查步骤：**
1. 打开「设置 → 屏幕使用时间 → 内容和隐私访问限制」
2. 确认「允许更改」中的以下项目已开启：
   - ✅ 麦克风
   - ✅ 语音识别
   - ✅ 键盘
3. 如果这些选项是灰色的，说明有系统限制
4. 联系设备管理员或关闭家长控制

---

### 🔧 方案 5：使用配置描述文件诊断（高级）

如果以上方法都无效，可能是系统级权限问题。

**步骤：**
1. 在 Xcode 中查看控制台日志
2. 筛选 "KeyboardVM" 关键词
3. 查找具体的错误信息：
   ```
   [KeyboardVM] 权限诊断:
   完全访问: ✅/❌
   语音识别: ✅/❌
   麦克风: ✅/❌
   
   [KeyboardVM] AVAudioSession error: ...
   [KeyboardVM] Error code: ..., domain: ...
   ```
4. 将完整日志发送给开发者

## 🔍 技术原因分析

### 为什么需要重启？

iOS 键盘扩展运行在独立的沙盒进程中：
- **键盘扩展进程** 在首次加载时会读取权限配置
- 修改「允许完全访问」只更新了配置文件
- 但**已运行的键盘进程不会立即重新读取**
- 需要通过重启设备来**强制终止所有键盘进程**
- 重启后，键盘进程会重新初始化并读取新权限

### AVAudioSession 错误 2003329396 含义：

这个错误代码对应 `kAudioSessionNotInitialized` 或权限被拒绝：
- 键盘扩展尝试访问麦克风
- 系统检查权限缓存
- 如果缓存中没有「完全访问」标记
- 系统拒绝 AVAudioSession 初始化
- 返回错误码 2003329396

## ⚠️ 已知限制

### 某些情况下键盘录音**不可用**：

1. **使用密码输入框时**
   - 系统为安全考虑会禁用第三方键盘
   - 自动切换回系统键盘

2. **锁屏状态下**
   - 键盘扩展无法访问麦克风

3. **低电量模式**
   - 可能限制后台权限

4. **企业设备 MDM 限制**
   - 企业配置可能禁止第三方键盘的完全访问
